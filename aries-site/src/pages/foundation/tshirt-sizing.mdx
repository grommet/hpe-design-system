## What is t-shirt sizing?

T-shirt sizing is a tokenized scale that maps familiar clothing sizes (`small`, `medium`, `large`, etc.) to specific pixel values for component sizing and spacing.

Prefer tâ€‘shirt size tokens over hard pixel values. If a new value is needed, report it so it can be added as a token.

## Benefits

- Creates consistent interfaces across applications
- Establishes common language between designers and developers
- Enables responsive scaling across screen sizes
- Maintains composability and scalability

## The scale

The sizing scale follows three core principles:

- **Proportional**: Each step maintains a natural ratio to other steps
- **Composable**: Each step divides into halves, thirds, quarters; combines from smaller steps
- **Flexible**: Subsets serve various use cases from dense to spacious layouts

Built from dual base units: `24px` (primary) and `16px` (supplementary). The primary and supplementary scales combine via doubling/halving to produce:

**Pixel progression:** 1, 2, 3, 4, 6, 8, 12, 16, 24, 32, 48, 64, 96, 128, 192, 256, 384, 512, 768, 1024, 1536

See [Layout and Spacing design tokens](/design-tokens/layout-and-spacing) for precise pixel values and implementation details.

## Composability

Sizes maintain mathematical relationships for predictable combinations.

```jsx
// Spacing sizes double:
xsmall (12px) + xsmall (12px) = medium (24px)
medium (24px) + medium (24px) = xlarge (48px)

// Container sizes follow the same pattern:
xsmall (192px) + xsmall (192px) = medium (384px)
medium (384px) + medium (384px) = xlarge (768px)

// Intermediate sizes fill gaps:
small (16px) = medium (24px) - xxsmall (8px)
large (32px) = medium (24px) + xxsmall (8px)
```

## Component Dimensions

Control component size using:
- `width` and `height` props ([Box](/components/box))
- `size` prop ([Avatar](https://v2.grommet.io/avatar?theme=hpe), [DataTable](/components/datatable), [Text](/foundation/typography#text))
- `row` and `column` dimensions ([Grid](/components/grid))

Refer to [Grommet documentation](https://v2.grommet.io/) for component-specific sizing props.

```
<Box width="medium" height="small" />
```

## Spacing

Control spacing using t-shirt sizes for `pad`, `gap`, `margin`, and `border`.

- Spacing (`edgeSize`): Controls pad, gap, and margin
- Border (`borderSize`): Controls border width

```
<Box

    height="small"
    pad="large"
/>
```

## Borders

- Use named border sizes to reinforce scale consistency.
- Avoid arbitrary pixel borders; Instead apply `border={{ size: 'small' }}`.

```
<Box 
  border={{ color: 'border', size: 'small' }}
  width="medium"
  height="small" 
/>
```

## Containers

- Limit width/height constraints to named container tokens to preserve responsive flow.
- Avoid unnecessary restriction; Instead favor [content-driven layouts](/templates/content-layouts#content-driven-layouts).
- Provide 11 container tokens for varied layout needs.

## Radius

- Use larger radius values for higher-level sections; Limit inner elements to smaller radii.
- Prevent inconsistent corner treatment; Always apply a named radius token.

## Default Size

- Recognize `medium` as the implicit `size` for components exposing `size`.
- Omit `size="medium"` to reduce noise when default behavior is desired.

## Responsiveness

Layouts should be designed with responsiveness in mind to meet a user wherever they are &mdash; whether on a desktop, tablet, or mobile device.

The values of semantic spacing and radius design tokens step down in size at the `small` [breakpoint](/design-tokens/global#breakpoint). This retains appropriate proportions between shape, spacing, and content as the screen size decreases.

When utilizing the CSS files, a media query is included in the `dimension.small.css` file to adjust the token values at the `small` breakpoint.

To further refine layouts for different screen sizes, various [breakpoints](/design-tokens/global#breakpoint) can be used to instruct logic for layout adjustments.
