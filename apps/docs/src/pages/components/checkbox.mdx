## Behaviors and states

### Interactive states

CheckBox responds to mouse and keyboard interaction:

- **Rest** — default appearance.
- **Hover** — visual indicator highlights when the cursor is over the control or label.
- **Focus** — focus ring appears when the control receives keyboard focus.
- **Active** — pressed appearance while the user holds the mouse button or Space key.

### Application states

#### Disabled

Apply `disabled` when the option is unavailable due to permissions or application-controlled conditions. Do not show a disabled CheckBox for an action the user lacks permission to perform permanently — hide the option instead.

## Content guidelines

- Start labels with a noun or verb that describes the choice clearly.
- Use sentence case; avoid title case and all-caps.
- Limit labels to a single line where possible.
- Frame labels as positive statements so checking confirms agreement or activation (e.g., "Receive notifications" not "Do not receive notifications").
- Write description text as a single sentence that expands on the label's meaning.
- For toggle labels, use a noun or noun phrase describing the setting (e.g., "Dark mode", "Email notifications").

## Accessibility

Always place CheckBox inside a FormField to ensure the label is programmatically associated with the input. When used outside a FormField, provide an accessible name via `a11yTitle` or `aria-label`.
