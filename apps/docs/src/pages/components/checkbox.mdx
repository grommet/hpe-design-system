## Introduction

CheckBox can be used to present an individual option or group of options to the user. When presented in a group, the user can select a single or multiple options.

## Use cases

### When to use

- Present an individual option or group of options to the user.
- Serve as a UI control showing or hiding page content (such as revealing additional content or inputs relevant only when a feature is enabled).
- Use in scenarios where providing a form field label in addition to the checkbox label would be redundant.
- Allow a user to opt-in or acknowledge a disclaimer, such as a privacy policy or terms and conditions.
- Allow an item to be selected within a list, table row, or drop-down.

### When to avoid

- Never use if a user is only allowed to select one option from a list of options. Instead, use `RadioButtonGroup`.

### With description

Adding a description provides the user additional information about the context or purpose of the checkbox. Use this when the user needs additional context about the checkbox's purpose.

### Toggle

Toggle is a single option which prompts users to choose between two mutually exclusive options. Use a toggle to quickly switch between two possible states, usually providing immediate results without form submission, allowing users to control their preference.

## Content Guidelines

- Always write labels that clearly describe the purpose of the individual checkbox.
- Always include clear visual indication, either by text or icon, that describes the purpose of the checkbox.
