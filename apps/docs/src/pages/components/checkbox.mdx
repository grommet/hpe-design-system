## Use cases

### Opting into an agreement or acknowledgement

Use CheckBox when a user must explicitly confirm consent, accept terms, or
acknowledge a disclaimer before proceeding. A single, clearly labeled
checkbox makes the action unambiguous.

<Example height={{ min: 'medium' }}>
  <CheckBoxOptingIntoAgreementExample />
</Example>

### Selecting one or more options from a list

Use CheckBox when a user may choose any number of items from a set of
options. Present each option as a labeled checkbox inside a FormField or
a CheckBoxGroup.

<Example height={{ min: 'medium' }}>
  <CheckBoxSelectingOptionsExample />
</Example>

### Controlling page content visibility

Use a standalone CheckBox outside a form to show or hide a section of
page content. The checkbox acts as a UI control, not a data input — no
form submission is required.

<Example height={{ min: 'medium' }}>
  <CheckBoxControllingContentVisibilityExample />
</Example>

### Switching a feature on or off with a toggle

Use the `toggle` variant when the setting takes effect immediately without
form submission. Toggles are preferred for preference controls where the
result is visible at once.

## Behaviors and States

### Interactive states

- **Rest** — default unchecked or checked appearance.
- **Hover** — subtle background change to indicate interactivity.
- **Focus** — visible focus ring for keyboard navigation.
- **Active** — pressed state during click or Space key activation.
- **Checked** — filled input control indicating the option is selected.
- **Unchecked** — empty input control indicating the option is not selected.

### Application states

- **Disabled** — prevents interaction; use when the option is unavailable due to permissions or a dependency.
- **Indeterminate** — used when the checkbox is a parent to child options and only a subset are checked. Set via the `indeterminate` prop.
- **Error** — surfaces a validation error via `FormField` `error`.
