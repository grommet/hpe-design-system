## Behaviors and States

### Interactive States

CheckBox provides rest, hover, focus, checked, and unchecked interactive states through the Grommet theme. No additional implementation is required.

### Application States

#### Disabled

Use `disabled` when the checkbox cannot be interacted with in the current context. Avoid using disabled checkboxes to convey unavailability due to permissions — hide the control entirely in those cases.

#### Indeterminate

Use the `indeterminate` prop when a parent checkbox has only a subset of its child options checked. Do not manually set `indeterminate` on standalone checkboxes — it is only meaningful in a nested selection group.

### General Behaviors

#### Using CheckBox outside of FormField

A FormField is required to properly associate the label with the input for accessibility. In cases where a FormField would be redundant, it is acceptable to use CheckBox without one:

1. Serving as a UI control showing or hiding page content.
2. Scenarios where having a FormField label in addition to the checkbox label would be redundant.
3. Allowing a user to opt in to or acknowledge a disclaimer.
4. Allowing an item to be selected within a list, table row, or dropdown.

When using CheckBox outside of a FormField, always ensure a clear label is visible or an accessible label is provided via `a11yTitle` or `aria-label`.

## Content Guidelines

- Write labels as concise phrases that clearly describe what enabling the option does.
- Use sentence case; avoid title case.
- Avoid negative phrasing (e.g., "Don't send me emails") — prefer positive framing ("Send me emails").
- Keep descriptions to one short sentence that adds context not already conveyed by the label.
- For toggle checkboxes, the label should describe the feature being controlled, not the on/off state itself.

## Accessibility

Always use CheckBox inside a FormField to ensure the label is properly associated with the input for assistive technologies.

When a CheckBox is used outside of a FormField, meet accessibility requirements by ensuring a clear visible label or by supplying `a11yTitle` or `aria-label` on the CheckBox. There must always be clear indication — either text or icon — describing the purpose of the checkbox.

### Keyboard interaction

| Key | Action |
| --- | --- |
| Space | Toggles the checked state. |
| Tab | Moves focus to the next focusable element. |
| Shift + Tab | Moves focus to the previous focusable element. |
