## Behaviors and states

### Interactive states

Menu supports the following interactive states on the trigger button: rest, hover, focus, and active. These states are handled automatically by the component and follow the HPE Design System's interactive state styles.

### Application states

#### Disabled

The menu trigger button can be set to a disabled state using the `disabled` prop. A disabled menu indicates that the actions exist but are not currently available, typically because a prerequisite condition has not been met. Avoid disabling a menu without communicating to the user why the actions are unavailable and what action is needed to re-enable them.

### General behaviors

#### Organizing menu items

Help users quickly locate the desired menu item by applying thoughtful ordering and grouping:

- Group related items together for easier scanning. Use a divider to separate groups.
- Prioritize frequently used actions by placing them at the top.
- Place destructive actions last in a distinct, consistent location to ensure intentionality.

#### Common groupings in HPE applications

In many HPE applications, customers configure, modify, and act upon resources in their environment. For these contexts, menu items typically fall into three groups:

**Configuring** — Actions allowing a user to set and modify attributes of a resource.

- View [resource] details
- Edit [resource]
- Add [item] to [resource]
- Remove [item] from [resource]
- Assign [resource] to [another resource]
- Add [resource] to group

**Executing** — Actions providing the ability for a user to act upon a resource.

- Launch [resource]
- Power on/off [resource]
- Reset [resource]
- Update/upgrade software/firmware version for [resource]

**Transferring** — Actions enabling a user to share or consume data related to a resource.

- Download [resource] data
- Share [resource] with [user]
- Send [resource] to [user]
- Notify [user] about [resource]

## Content guidelines

- Give the menu a clear, descriptive title that conveys its purpose rather than a generic label such as "Menu." For example, use "Account information" for a menu containing actions like "Change username" or "Reset password."
- Keep menu item labels short and action-oriented using simple, concise wording — for example, "Settings" or "Logout" — so users immediately understand the action that will be performed when they click.

## Accessibility

Accessibility features such as screen reader messages are automatically built into Menu. By default, these messages are "Open Menu" and "Close Menu." If a more specific message is needed, apply a custom [`aria-label`](https://v2.grommet.io/menu?theme=hpe#a11yTitle). For internationalization use the [`messages`](https://v2.grommet.io/menu?theme=hpe#messages) prop.

Always provide an `aria-label` for icon-only menus so screen reader users understand the button's purpose.

### Keyboard support

| Key                         | Action                                                                       |
| --------------------------- | ---------------------------------------------------------------------------- |
| **Enter / Space**           | Opens or closes the menu.                                                    |
| **Escape**                  | Closes the menu and returns focus to the trigger button.                     |
| **Arrow Down / Arrow Up**   | Moves focus between menu items when the menu is open.                        |

### ARIA

| Attribute        | Value          | Condition                                                                        |
| ---------------- | -------------- | -------------------------------------------------------------------------------- |
| `aria-haspopup`  | `true`         | Always present on the trigger button.                                            |
| `aria-expanded`  | `true / false` | Reflects whether the drop container is currently open or closed.                 |
| `aria-label`     | descriptive string | Applied when the trigger button uses an icon only with no visible label text. |
