## Variants

### Primary button

High-contrast, filled button used for the single most important action on a page.

Use when there is one clear, high-value action available (e.g., Create, Deploy, Save). Only one primary button should appear per view.

### Toolbar button

Flat button with equal visual weight, designed to sit alongside other toolbar controls.

Apply `kind="toolbar"` when the button appears as part of a row of toolbar controls — for example, a search input, a filter button, and an action menu — where each control should share equal prominence. Apply `kind="toolbar"` to Button or Button-based components such as Menu.

### Button with badge

The `badge` prop adds a small indicator to a button to convey extra information.

- Pass `true` to display a dot when something requires attention.
- Pass a number to display a count (e.g., 3 new messages). The badge caps at the `max` value (default 9) and appends "+" when exceeded (e.g., 9+).

### Application states

#### Disabled

Prevent interaction by setting `disabled`. Prefer leaving buttons active and displaying a contextual message on click when criteria have not been met. Never show a disabled button for actions unavailable due to permissions or context — hide it entirely instead.

#### Busy

Use `busy` to indicate an in-progress async operation triggered by the button. Apply when a short server-side validation (a few seconds) is underway. For operations expected to take longer, release the button and notify the user separately (e.g., via a notification).

## Content guidelines

### Button labels

- Use clear and concise wording.
- Start labels with a verb that describes the action (e.g., "Save changes", "Delete device").
- Use sentence case; avoid title case.
- Avoid contractions or colloquialisms; keep language approachable but not personal.
- Limit labels to 2–4 words where possible.
- Ensure the label matches the route destination or resulting action.

### Add, New, and Create

| Label | Purpose | Example |
| --- | --- | --- |
| **Add** | Builds an association between an existing object and another. | "Add device" when adding an existing device to a group. |
| **New** | Opens a blank template for user input before creation. The object is not persisted until a "Create" or submission action completes. | "New device" to initiate configuration of a new device. |
| **Create** | Submits user input and persists the object as a record. | "Create device" after completing configuration inputs. |

### Button alignment

Buttons are unique because their alignment depends on context (drawers, modals, pages, etc.). Always position buttons where users most expect them.

### Buttons vs. anchors

Select the correct HTML element based on the resulting action:

- Use a **link** when navigating to another URL (e.g., a "View all" page, a user profile, a skip link).
- Use a **button** when performing an action (e.g., submit, merge, create, upload). Actions almost always occur on the same page.

### Toggle button labels

- Limit the accessible label to 2–3 words.
- Start with a verb (e.g., "Hide menu", "Favorite item", "Switch theme").
- Use sentence case.
- Avoid articles such as "the" or "a".

### Cancel buttons

Use default button styling for cancel buttons. The neutral appearance signals a non-committal action. Avoid visually emphasizing cancel to prevent user confusion.

### Responsiveness

- Ensure button labels never wrap, truncate, or scale down on smaller browser widths.
- Maintain a minimum button width of 42px.
- On screen widths ≤ 576px, stretch single buttons to fill the full container width.
- Stack button groups vertically on screen widths ≤ 576px.

## Accessibility

If activating a button results in a new URL or a UI state that makes sense as a browser tab, use a link (`<a>`) instead of a button (`<button>`). Screen readers announce these elements differently; choosing the wrong element misleads users about what will happen next.

For icon-only buttons, always supply `a11yTitle` (Grommet prop) or `aria-label` to provide an accessible name.

For toggle buttons with icons, show the current state via an icon and communicate the resulting state via `a11yTitle` or `aria-label`.

### Keyboard support

| Key | Action |
| --- | --- |
| `Space` | Activates the button. |
| `Enter` | Activates the button. |
| `Tab` | Moves focus to the next focusable element. |
| `Shift + Tab` | Moves focus to the previous focusable element. |
