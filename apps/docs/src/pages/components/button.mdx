## Introduction

Button is an interactive element that enables users to submit forms, begin tasks, trigger UI changes, or specify next steps in a process.

## Use cases

Use buttons to:

- Submit a form
- Begin a new task
- Trigger a new UI element to appear on the page
- Specify a new or next step in a process
- Perform an action on the same page (e.g., submit, merge, create new, upload)

Avoid using buttons when:

- Navigating to another place. Instead, use an anchor or link.

## Types

### Primary

Primary buttons highlight actions on a page delivering the most benefit to users. They have increased visual weight through placement and contrast.

Use for the key action on the page. Never use more than one primary button per page.

### Secondary

Secondary buttons represent calls to action that either support the primary call to action or stand alone.

Consider that the 80% use case for some pages is visualization and not action. Use a secondary button for these cases. Include multiple secondary buttons per page as needed.

### Cancel

Cancel buttons enable users to exit out of or dismiss the current screen in order to go back to the previous one.

Provide an explicit way to cancel as a fallback to safety to discard unwanted changes. Always use default button styling.
