## Introduction

Buttons should be used in situations where users might need to:
- Submit a form
- Begin a new task
- Trigger a new UI element to appear on the page
- Specify a new or next step in a process

## Use cases

### When to use
- Use buttons when performing an action, such as "submit", "merge", "create new", or "upload".
- Always place actions on the same page.
- Highlight actions on a page delivering the most benefit to users.

### When to avoid
- Never use a link/anchor when performing a primary action.
- Never use a button when navigating to another place, such as a "view all" page, profile, or "skip link" (use an anchor).
- Never show inapplicable buttons as disabled; remove them completely from the UI instead.

## Variants

### Primary
Highlights actions on a page delivering the most benefit to users, unlocking new value by enabling a new experience or creation.
Always use for the key action on the page. Never include more than one primary button per page.

## Content Guidelines

- Always apply an appropriate label so the user has a clear understanding of what action to expect.
- Use clear and concise wording.
- Never use contractions or colloquialisms. Keep the language approachable but not personal.
- Always ensure your button label matches the route destination.
- Limit icon buttons to 2-3 descriptive words and use verbs first (e.g. "Hide/View", "Switch/Unswitch", "Favorite/Unfavorite").
- Always use sentence case capitalization. Never use words such as "the" and "a" in button labels.
