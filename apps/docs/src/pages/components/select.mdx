## Use cases

### Choosing a single value from a predefined list

Use Select inside a Form when the user must choose one option from a fixed set of values and screen space must be conserved. Options stay hidden until the user opens the dropdown — reducing visual footprint compared to inline radio buttons.

Use Select with the `onSearch` prop when presenting a lengthy options list such as countries or regions. Type-ahead filtering narrows the list without requiring the user to scroll.

## Behaviors and states

### Interactive states

- **Rest** — the Select control is visible but no user interaction is occurring.
- **Hover** — the cursor moves over the control; a visual hover style is applied.
- **Focus** — the control receives keyboard focus; a focus ring is applied.
- **Active** — the user is actively pressing the control.
- **Open** — the dropdown is expanded and the option list is visible.

### Application states

#### Disabled

Disable Select when interaction is not yet permitted. Apply the `disabled` prop to both the `Select` and its enclosing `FormField`.

#### Validation

Validation styling highlights the Select input with an error message. Always accompany validation errors with text that tells users how to resolve the issue. Apply validation through the `FormField` `error` prop. A required-field error is the most common pattern since available options are predefined.

### Data states

- **Empty** — no value is selected; placeholder text is displayed in the control.
- **Has value** — a value is selected and displayed in the control.

## Content guidelines

- Use concise, descriptive labels for each option; avoid generic values such as "Item 1" or "Option A".
- Write option labels in sentence case.
- Use a placeholder that describes the expected selection (e.g., "Select a region") rather than generic text like "Select...".
- Always pair Select with a FormField label to communicate the purpose of the input to all users.

## Accessibility

Prefer using Select within a `FormField`. This gives users clear indication of the input's purpose and ensures labels accompany inputs for assistive technologies.

Always give the Select an `id` and set `htmlFor` on `FormField` to `${id}__input` so the label associates correctly.

```
<FormField label="Select option" htmlFor="select__input">
  <Select id="select" options={['Option 1', 'Option 2']} />
</FormField>
```

### Keyboard interactions

| Key                   | Action                                                         |
| --------------------- | -------------------------------------------------------------- |
| Space / Enter         | Opens the dropdown when the control is focused                 |
| Escape                | Closes the dropdown and returns focus to the trigger control   |
| Arrow Down / Arrow Up | Moves focus between options when the dropdown is open          |
| Enter                 | Selects the focused option and closes the dropdown             |
| Tab                   | Moves focus to the next focusable element; closes the dropdown |

### ARIA attributes

| Attribute         | Value                       | Condition                                                                  |
| ----------------- | --------------------------- | -------------------------------------------------------------------------- |
| `aria-expanded`   | `true` / `false`            | Reflects whether the option list is open or closed                         |
| `aria-haspopup`   | `listbox`                   | Always present                                                             |
| `aria-labelledby` | FormField label element id  | When paired with FormField via `htmlFor="${id}__input"` on FormField       |
