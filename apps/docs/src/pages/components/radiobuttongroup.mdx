## Use cases

### Selecting a single configuration option

Use RadioButtonGroup when a form asks the user to choose exactly one value from a small, fixed set — for example, a power regulation mode or a network policy. All options remain visible so the user can compare them at a glance before deciding.

## Behaviors and States

### Application States

#### Validation

Validation styling indicates a required selection was not made. Provide a clear group label and a specific error message. Where possible, explain why the selection is important to the user — not just "required."

## Content Guidelines

- Keep option labels concise and mutually exclusive in meaning.
- Use parallel grammatical structure across all options — e.g., all noun phrases or all verb phrases.
- Avoid negations in option labels (e.g., "Do not apply settings") — rephrase as a positive.
- Write the group label as a clear question or noun phrase describing what is being chosen.
- Limit the group label to one short phrase; avoid full sentences.

## Accessibility

### Keyboard interaction

| Key | Action |
| --- | --- |
| Tab | Moves focus into the RadioButtonGroup; focuses the currently selected option or the first option if none is selected. |
| Arrow Down / Arrow Right | Moves focus to and selects the next radio button in the group. |
| Arrow Up / Arrow Left | Moves focus to and selects the previous radio button in the group. |
| Space | Selects the focused radio button. |

### ARIA

- The RadioButtonGroup container has `role="radiogroup"` applied automatically.
- Apply `required` on FormField to set `aria-required="true"` on the control when selection is mandatory.
- When a validation error is present, `aria-invalid="true"` is applied automatically through FormField.
- Always provide an accessible group label via the FormField `label` prop, or directly via `a11yTitle` when no visible label is present.
