# Design System Playground - Working Reference

This folder is the single source of truth for the playground build as it evolves.

## How to use this reference
- Start with the primer and scope in [primer.md](primer.md).
- Track decisions and changes in [decisions.md](decisions.md).
- For each audited component, add a brief summary in [components/README.md](components/README.md) and keep JSON outputs in [components/].
- Capture cross-component rules in [patterns.md](patterns.md).
- Record open questions and risks in [open-questions.md](open-questions.md).

## Canonical artifacts
- Component prop audit JSON: [components/]
- Prop schema definition: [schemas/prop-audit-schema.json](schemas/prop-audit-schema.json)
- UI control mapping rules: [ui-controls.md](ui-controls.md)
- Build architecture notes: [architecture.md](architecture.md)
